Wheel of Fortune MVP (built 2026-06-26). Daily free spin at the Town Hub for a rarity-weighted random prize.

**Data** — `ReplicatedStorage.GameData`: `WheelPrizes` (8 slices, each with rarity/label/weight/kind), `WheelCooldown=24*3600`, `WheelRarityColor` map, `WheelTotalWeight()`, `WheelRoll(rng)` (server-only weighted roll). Slice kinds: `gold` (AddGold), `crystal` (AddCrystal crystalId), `item` (GrantItem itemId, with `fallbackGold` if already owned). MVP pool = gold tiers + Forest crystals + LollipopHammer (Epic) + 2000g jackpot (Legendary, weight 1/100). Weights sum to 100 so weight = exact %.

**Persistence** — `ProfileService`: `wheelLastSpin` field (os.time, saved/loaded). `WheelCooldownRemaining(plr)`, `CanSpinWheel(plr)`, `DoWheelSpin(plr)` → rolls, stamps cooldown BEFORE granting, returns (ok, idx, prize, rewardText).

**Server** — `ServerScriptService.WheelService`: builds the `WheelKiosk` model in `Workspace.TownHub.Services` at (820,?,0) (open +X side of plaza; pad `WheelPad` has the ProximityPrompt "Spin the Wheel"). Handles `WheelSpin` remote (debounced), pushes `WheelState` on join + after spins. Kiosk is rebuilt each server start.

**Client** — `StarterPlayer.StarterPlayerScripts.WheelUI`: prompt opens a centered panel; wheel = 8 colored "petal" frames + spokes + baked labels; spins via TweenService to land the server-chosen slice under the top pointer; shows reward text + live cooldown countdown. Remotes: `WheelState`, `WheelSpin`.

**Gotchas learned this build (Roblox UI):** (1) `GuiObject.Rotation` ALWAYS pivots about the element's own centre, NOT its AnchorPoint — position rotated radial elements with cos/sin trig, then rotate. (2) `NumberSequence` MUST end at time=1.0 or it throws and aborts the script. (3) Pure-frame pie wedges via ClipsDescendants do NOT work (UICorner doesn't clip children; rotated-clip half-plane intersection fails) — used radial colored petals + spokes instead. (4) `screen_capture` in Play only reliably shows a ScreenGui layer when its backdrop is opaque. A perfect conic wheel PNG was generated but couldn't be uploaded (sandbox↔Windows file sync lag broke store_image/upload_image).

Prize pool is intentionally easy to extend/rebalance — just edit `GameData.WheelPrizes`. Related: [[project_town_hub_build]], [[project_rebirth_shop_build]], [[game_balance_theorycraft]].

## 2026-07-07 visual redesign (Claude Design mockup)

User designed a new wheel look in Claude Design (HTML/CSS mockup, saved to the shared repo as `Wheel UI.html`) and asked to integrate it. Restyled `WheelUI` (Studio MCP bridge, direct edit) — spin/prize CONTRACT unchanged (WheelState/WheelSpin remotes, WheelPrizeInfo resolver, server-authoritative land). Changes: (1) **slice colour is now a fixed decorative 8-colour palette by index** (`WHEEL_PALETTE`: green 95,191,102 / blue 87,174,242 / green 73,176,87 / purple 110,81,181 / yellow 242,198,63 / green 55,164,95 / red 238,90,80 / light-green 116,220,130), REPLACING the semantic kind/rarity `sliceColor` scheme — prize meaning still read from each slice's LABEL + ICON, so no info lost. The per-crystal petal recolour (in refreshLabels) and the astral-mystery gradient/star override are DISABLED (guarded `if false`, kept as dead code) so the palette stays stable. (2) Backing disc darkened to near-black **(18,10,5)** (was 64,42,30). (3) Petals made **flat solid** — the white→col gloss `UIGradient` was neutralised (const white, transparency 0) because over the near-black disc its transparency-fade read as muddy dark. (4) Petals widened to **0.32 × 0.46** (PETAL_H 0.46) so they tile into a bold octagon/pinwheel that fills the disc. (5) Pointer → bright red **(253,29,29)** with a white **UIStroke** outline (thickness 3), TextStroke off. (6) Hub bumped to 0.2 scale, gold rim thicker (thickness 6, colour 255,206,59); kept the "$" glyph. Verified live in Studio (Play): all 8 palette colours confirmed on the petals, spin lands the server-chosen slice under the pointer, reward text shows, no script errors (only the benign StudioAccessToApisNotAllowed DataStore warning). To revert to semantic colours: use `sliceColor(prize)` in the petal loop and re-enable the two `if false` guards.

**Overflow fix — true pie wedges (same day, user feedback).** The palette pass above kept the old rounded "petals" (widened to 0.32×0.46) — but they bulged OUT past the black rim; user compared to the mockup and flagged the overflow. EditableImage (the clean conic route) is gated behind the place's Security "Editable Image/Mesh API" toggle (WritePixelsBuffer → "EditableImage is not accessible"), so it was avoided. Final = asset-free TRUE sectors: per 45° slice, a container Frame rotated to `s*SLICE_DEG` (children rotate WITH the parent in Roblox) holds `WEDGE_RES=42` stacked radial bars (a rasterised triangle), bar width `2*rm*tan(SLICE_DEG/2)`, built out to `WEDGE_R=0.455` along the centre-line so the outer CORNERS land at r≈0.49 — inside the 0.5 disc (a triangle's corners sit at R/cos(22.5°), so WEDGE_R must be < 0.462; verified numerically maxCornerRadius 0.512→0.492). Old petals hidden via `p.Visible=false` (kept for the jackpot-rim ref); gold dividers thickened to 0.016×0.47 to mask the stepped seams. LIMITATIONS (cosmetic, not yet done): outer edge is a straight chord (octagon) not the mockup's rounded arc, and the hub is still the green "$" coin not the mockup's multicolour pinwheel.
